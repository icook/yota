Yota

Yota is the product of personal aggrivation with overly complicated python form libraries that always seemed to get in my way. My goal is to create something that is flexible and non-imposing on style. Yota also aims to maintain a tiny, well documented code base, making it easy to contribute and tweak for your own needs.

Regular Forms

Below is a simple example form generated with Yota. The form is re-rendered on submission via Form.render_validate() to provide automatic injection of validation errors into the templating context. This injection is provided via a callback method in the Node class that can easily be overridden if a different semantic is preferred.

AJAX Submit Forms

The form below uses very similar code to the above example except a flag is passed into the global rendering context designating it as an AJAX form. This triggers the inclusion of Javascript snippets and placeholder error divs that enable transparent validation. Validation messages are populated by callbacks defined in Javascript and are supplied with all the data that the Python validator returned. Although a general pattern is for the validator to return a message attribute in practice the system is flexible and can return anything.
